## 3. 候选方向（需主人决策）

### 方向 A：稳定收尾 + 可观测增强（保守）
专注把当前 V21 的问题修完，加上可观测性改进，不引入大新功能。

**核心交付：**
- V22-M1：响应式验证 + TODAY 盒最终版 + 活跃时长统计最终版
- V22-M2：Scene Event 可读性——每个事件在画布上有轻量文字标注（当前事件类型 + 参与者）
- V22-M3：Inspector 增强——展示当前 persona type 和具体行为描述，不只是状态
- V22-M4：Token 趋势图——右侧面板 TODAY 盒下方加 mini sparkline（每5分钟采样）

**优点：** 稳健，风险低，每阶段可独立交付
**缺点：** 没有突破性改进

---

### 方向 B：Monitor 健康监控（功能性）
利用现有后端采集能力，做真正的监控告警功能。

**核心交付：**
- V22-M1：Monitor Health 指标——collector 采集延迟、API 响应时间、后端错误率
- V22-M2：告警规则——消息/Token 超阈值、状态异常持续太久
- V22-M3：告警历史——记录哪些时刻出现过告警
- V22-M4：飞书 WebHook 告警通知（复用现有飞书 Bot）

**优点：** 实用，真正解决"不知道服务挂了"的问题
**缺点：** 需要新增数据库表，飞书凭证依赖

---

### 方向 C：预测系统集成（智能体方向）
将大乐透预测系统接入监控面板，作为 V22 特色模块。

**核心交付：**
- V22-M1：预测面板——显示最近预测号码、预测时间、投入金额
- V22-M2：开奖结果录入——自动/手动录入开奖号码，标注命中
- V22-M3：命中率统计——大奖命中率、缩小范围命中率
- V22-M4：预测历史 + 趋势图

**优点：** 与大乐透预测系统联动，形成完整闭环
**缺点：** 需要协调两个项目，大乐透系统本身还在规划中

---

## 4. 推荐方案

**推荐方向 A（稳定收尾 + 可观测增强）**

理由：
1. V21-M5 刚发布，需要一个稳定收尾版本而不是立刻跳到新功能线
2. 主人多次强调"方案理清楚之前不进行任何改动"，方向 A 风险最低
3. Scene Event 可读性和 Inspector 增强是用户明确表示需要的
4. Token 趋势图在现有数据结构下实现成本低
5. 方向 B/C 都需要新增数据库或跨系统集成，不适合作为 V22 第一个阶段
